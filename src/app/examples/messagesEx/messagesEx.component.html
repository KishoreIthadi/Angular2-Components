Component Code

This component comes up with MessagingService & MessagingComponent. It is better to declare the messaging component at root level in the main module. Where ever you need the messages to be shown, Inject MessagesService into your component constructor and use it as shown below.

The MessagesService have set method which takes two arguments message and CSS class. By default, the component comes with four classes messages_success, messages_danger, messages_warning & messages_info. You can pass your own classes and override the existing styles. 

    
        constructor(private messagesService: MessagesService){ } 
        
        click() {
        this.messagesService.SetMessage("Success Message", "messages_success");
        this.messagesService.SetMessage("Warning Message", "messages_warning");
        this.messagesService.SetMessage("Error Message", "messages_danger");
        this.messagesService.SetMessage("Info Message", "messages_info");
        }
